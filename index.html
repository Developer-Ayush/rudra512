How It Works
// Rudra-512 v2 processes input in 64-byte blocks through six hardened phases
STEP 01
Salt Prefix Encoding
Salt encoded with 4-byte BE length + salt bytes + 0x01 separator. Absent salt → single 0x00 byte. Empty salt ≠ absent salt.
// salt present: [len_BE4][salt][0x01] // no salt: [0x00]
STEP 02
Rounds Domain Binding
Rounds encoded into initial state before first block. hash(x,r1) and hash(x,r2) diverge structurally from byte 1.
tweak = rounds * π_frac s[0] ^= tweak s[7] ^= rotl(tweak, 32)
STEP 03
Merkle-Damgård Padding
Input padded to 64-byte multiple. 0x80 marker, zero fill, then 64-bit BE message length. Single stream engine for all inputs.
msg || 0x80 || 0x00... || len64_BE
STEP 04
Non-linear Absorption
State-dependent tweak: s[j] placed inside rotl argument — algebraically non-cancellable. Prevents meet-in-the-middle and state-erasure attacks.
tweak = rotl(idx+j+s[j], ROT[j%8]) s[j] ^= word ^ tweak
STEP 05
Snapshot Permutation
Full state snapshot before each round eliminates sequential-update asymmetry. ROUND_ROT[(i×5+r×9)%64] gives period-64 rotation schedule per word.
old = s[:] v = old[i]^rotl(old[(i+mix)%8],ROT[i]) v += old[(i+mix2)%8] v = rotl(v, ROUND_ROT[(i*5+r*9)%64]) v ^= old[(i+5)%8] >> 17 s[i] = v // then cross-mix pass
STEP 06
Finalisation
After all blocks absorbed: one final permutation for the full round count (min 1). State words serialised as 16-char zero-padded hex.
permute(state, rounds) digest = hex(s[0..7]) // 128 chars
